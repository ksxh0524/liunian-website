# 设计分析报告：Notion 与 Cursor 官网设计研究

**调研日期**：2026-03-07  
**调研对象**：Notion (https://notion.so) / Cursor (https://cursor.sh)  
**调研目的**：为流年官网（AI 聊天记忆系统）提供设计参考

---

## 一、Notion 官网设计特点

### 1. 布局结构

**核心特征**：
- **Bento Grid 风格**：大量使用圆角卡片网格布局，每个功能模块独立成块
- **分层信息架构**：Hero 区域 → 功能展示 → 社会证明 → CTA，层次清晰
- **留白充足**：模块间距大（约 80-120px），视觉呼吸感强
- **不对称平衡**：文字与图形/截图交替左右排列，避免单调

**具体表现**：
```
Hero 区域：
- 大标题 + 副标题 + 双 CTA 按钮
- 背景使用抽象动态图形（彩色圆点 + 曲线）
- 底部信任标识（98% of Forbes Cloud 100）

功能展示区：
- 采用卡片网格，每个卡片展示一个功能
- 卡片内含：图标 + 标题 + 简短描述 + 箭头链接
- 部分卡片带有交互演示（如 Custom Agents 轮播）
```

### 2. 动画效果

**观察到的动画**：
- **微交互动画**：按钮 hover 时轻微缩放（scale 1.02-1.05）
- **卡片翻转**：部分功能卡片展示前后两面（Docs、Projects）
- **轮播动画**：Custom Agents 展示区支持左右滑动切换
- **背景动态**：Hero 区域有缓慢移动的抽象图形
- **数字滚动**：统计数据（100M users）有滚动计数效果

**动画特点**：
- 缓动函数：cubic-bezier(0.4, 0, 0.2, 1)
- 持续时间：200-400ms
- 触发方式：scroll-based + hover + auto-play

### 3. 交互设计

**交互亮点**：
- **计算器组件**："More productivity. Fewer tools." 区域的成本计算器
  - 可勾选多个服务选项
  - 实时计算团队规模和节省金额
  - 数字输入支持增减按钮
- **可切换 Tab**：Custom Agents 区域支持点击切换不同代理类型
- **视频预览**：客户案例卡片悬停时播放短视频
- **平滑滚动**：锚点链接滚动带缓动效果

### 4. 视觉层次

**层次构建方式**：
```
层级 1（最重要）：Hero 大标题 - 48-64px 字重 700
层级 2：功能卡片标题 - 24-32px 字重 600
层级 3：正文描述 - 16-18px 字重 400
层级 4：辅助信息 - 14px 字重 400, 透明度 60%
```

**色彩策略**：
- 主色调：黑白灰（#000, #333, #666, #999, #FFF）
- 强调色：彩色卡片背景（红#FF6B6B、蓝#4ECDC4、绿#45B7D1、黄#FFE66D）
- 渐变使用：轻微线性渐变增加深度

### 5. 文字排版

**字体系统**：
- 主字体：Notion 自定义字体（类似 Inter/SF Pro）
- 字重层级：300（Light）、400（Regular）、500（Medium）、600（SemiBold）、700（Bold）
- 行高：标题 1.1-1.2，正文 1.5-1.6

**排版特点**：
- 大标题使用 Tight Tracking（-0.02em）
- 段落最大宽度 65ch（约 65 字符）
- 列表项间距 12-16px

### 6. 图标和图形使用

**图标系统**：
- 风格：线性图标 + 填充图标混合
- 尺寸：16px、20px、24px 三档
- 使用场景：功能卡片、导航、CTA 按钮

**图形元素**：
- 抽象背景：Hero 区域使用彩色圆点 + 曲线
- 产品截图：圆角矩形，带轻微阴影
- 客户 Logo：单色处理（黑色或深灰色）

## 二、Cursor 官网设计特点

### 1. 布局结构

**核心特征**：
- **单栏叙事流**：从上到下的线性叙事，类似产品文档
- **代码优先**：大量使用代码编辑器截图/演示
- **对比展示**：传统方式 vs Cursor 方式并排对比
- **引用突出**：名人/CEO 推荐语使用大字号引用块

**具体表现**：
```
Hero 区域：
- 超大标题（56-72px）
- 简短副标题
- 单一 CTA（Download for macOS）
- 右侧/下方产品界面演示

功能展示：
- 左侧文字说明 + 右侧产品截图
- 交替布局避免单调
- 每个功能独立成段，间距 120-160px
```

### 2. 动画效果

**观察到的动画**：
- **代码打字机效果**：Hero 区域代码演示有打字动画
- **渐变流动**：背景有缓慢移动的渐变色
- **悬停高亮**：代码语法高亮在悬停时增强
- **淡入滚动**：元素进入视口时淡入（fade-in-up）
- **Tab 切换**：产品演示 Tab 切换带滑入动画

**动画特点**：
- 代码动画：逐字符显示，50-100ms/字符
- 滚动动画：Intersection Observer 触发
- 持续时间：300-600ms

### 3. 交互设计

**交互亮点**：
- **交互式代码演示**：Hero 区域可点击查看不同代码示例
- **模型选择器**：展示支持的各种 AI 模型（GPT-5、Opus、Gemini 等）
- **Tab 切换**：功能演示支持 Tab 切换不同场景
- **下拉菜单**：导航栏支持展开子菜单

### 4. 视觉层次

**层次构建方式**：
```
层级 1：Hero 标题 - 56-72px 字重 700
层级 2：功能标题 - 32-40px 字重 600
层级 3：引用文字 - 24-28px 字重 400, 斜体
层级 4：正文 - 16-18px 字重 400
层级 5：代码 - 14px 等宽字体
```

**色彩策略**：
- 主色调：米白/浅灰背景（#F5F5F0、#FAFAF8）
- 文字色：深灰（#1A1A1A、#333333）
- 强调色：橙色 CTA 按钮（#FF6B35）
- 代码主题：浅色语法高亮

### 5. 文字排版

**字体系统**：
- 主字体：Inter / SF Pro Display
- 代码字体：JetBrains Mono / Fira Code
- 字重：400、500、600、700

**排版特点**：
- 标题字间距正常，不紧缩
- 引用块使用斜体 + 大字号
- 代码块带行号和语法高亮

### 6. 图标和图形使用

**图标系统**：
- 风格：简洁线性图标
- 使用场景：功能列表、导航

**图形元素**：
- 产品截图：带阴影的圆角矩形
- 抽象背景：渐变色块 + 波浪线条
- 人物引用：头像 + 姓名 + 职位

## 四、设计趋势总结

### 2026 SaaS 官网设计趋势

1. **AI 原生视觉语言**
   - 使用 AI 对话界面作为核心视觉元素
   - 展示 AI 思考过程（typing indicator、thought bubble）

2. **极简色彩 + 局部强调**
   - 黑白灰为主，单色强调（橙/蓝/紫）
   - 避免过多色彩干扰

3. **交互式演示优先**
   - 可操作的产品演示 > 静态截图
   - 让用户在官网体验核心功能

4. **社会证明多样化**
   - Logo 墙 → 具体数据 → 名人引用 → 客户案例

5. **微动画提升质感**
   - 不夸张，但有存在感
   - 缓动曲线精细调校
